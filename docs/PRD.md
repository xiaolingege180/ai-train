# AI 工具培训平台 - 需求规格说明书 (PRD)

## 1. 项目概述

### 1.1 项目背景
随着人工智能技术的快速发展，越来越多的普通人需要学习和使用 AI 工具（如 ChatGPT、Midjourney、Claude 等）来提升工作效率。然而，目前缺乏系统性、互动式的学习平台来帮助零基础用户入门。

### 1.2 项目目标
开发一个 Web 应用平台，通过结构化课程、实践练习、AI 辅导和社区互动，帮助普通人系统性地掌握主流 AI 工具的使用方法。

### 1.3 目标用户
- 职场新人（22-28岁）：希望提升办公效率
- 中年转型者（35-45岁）：需要学习新技能适应变化
- 自由职业者：希望用 AI 提升内容创作效率
- 学生群体：对 AI 工具感兴趣，希望提前掌握技能

## 2. 功能需求

### 2.1 用户系统

#### 2.1.1 用户注册与登录
- 支持邮箱注册/登录
- 支持手机号注册/登录
- 支持第三方登录（微信、QQ、GitHub）
- 忘记密码功能
- 邮箱验证机制

#### 2.1.2 用户画像
- 注册时收集：职业、AI 基础水平、学习目标
- 根据画像推荐个性化学习路径
- 用户可修改个人资料和头像

#### 2.1.3 学习进度追踪
- 记录课程完成进度
- 记录练习得分和历史
- 生成学习报告和证书

---

### 2.2 课程系统

#### 2.2.1 课程分类
| 分类 | 描述 | 目标工具 |
|------|------|----------|
| 文本生成 | 文案、报告、邮件写作 | ChatGPT、Claude、文心一言 |
| 图像生成 | 海报、插画、设计 | Midjourney、Stable Diffusion、DALL-E |
| 代码辅助 | 编程学习、代码生成 | GitHub Copilot、通义灵码 |
| 办公效率 | PPT、Excel、文档处理 | AI Office 助手、Notion AI |
| 视频创作 | 脚本、剪辑、配音 | 剪映 AI、Sora、Runway |

#### 2.2.2 课程结构
- **章节**：每门课程包含 5-10 个章节
- **课时**：每个章节包含 3-5 个课时（视频/图文/互动）
- **知识点**：每个课时可标记多个知识点
- **难度等级**：入门 → 进阶 → 精通

#### 2.2.3 课程内容形式
- 视频教程（带字幕和倍速播放）
- 图文教程（支持代码高亮、图片放大）
- 互动演示（嵌入 AI 工具界面模拟器）
- 案例实战（真实场景任务）

---

### 2.3 实践练习系统

#### 2.3.1 Prompt 练习器
- 提供 Prompt 编写练习
- AI 实时评估 Prompt 质量
- 提供优化建议
- 保存优秀 Prompt 到个人库

#### 2.3.2 任务挑战
- 每章节配套实践任务
- 任务类型：
  - 选择题（概念理解）
  - 填空题（Prompt 补全）
  - 实操题（提交作品/结果）
  - 项目题（综合应用）
- AI 自动评分 + 人工审核

#### 2.3.3 AI 辅导助手
- 集成 LLM API（OpenAI/Claude/文心）
- 学习过程中随时提问
- 根据当前课程内容上下文回答
- 不直接给答案，引导思考

---

### 2.4 社区系统

#### 2.4.1 讨论区
- 按课程/章节划分板块
- 发帖、回帖、点赞、收藏
- 支持 Markdown 和代码块
- 图片/附件上传

#### 2.4.2 作品展示
- 用户分享学习成果
- 支持图片、链接、文字描述
- 他人可评论和点赞
- 优秀作品置顶推荐

#### 2.4.3 学习小组
- 创建/加入学习小组
- 小组内共享笔记和资源
- 组内学习排行榜

---

### 2.5 资源库

#### 2.5.1 Prompt 模板库
- 按场景分类：写作、编程、设计、办公
- 支持复制、收藏、自定义修改
- 用户可提交自己的 Prompt

#### 2.5.2 工具推荐
- AI 工具导航（分类+评分+简介）
- 官方文档链接
- 使用教程汇总
- 用户真实评价

#### 2.5.3 学习资料
- 电子书下载
- 速查表（Cheatsheet）
- 行业报告和白皮书

---

### 2.6 管理后台

#### 2.6.1 内容管理
- 课程 CRUD
- 章节和课时管理
- 题目和练习管理
- 资源上传管理

#### 2.6.2 用户管理
- 用户列表和详情
- 学习数据看板
- 消息推送
- 封禁/解禁

#### 2.6.3 数据统计
- 注册/活跃用户数
- 课程完成率
- 练习正确率分布
- 用户留存率

## 3. 非功能需求

### 3.1 性能要求
- 首屏加载时间 < 2s
- 视频加载时间 < 3s
- API 响应时间 < 500ms（P95）
- 支持 1000+ 并发用户

### 3.2 兼容性
- 浏览器：Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 移动端：响应式布局，支持 iOS Safari 和 Android Chrome
- 分辨率：PC 端 1280x720 以上，移动端 375x667 以上

### 3.3 安全要求
- 密码加密存储（bcrypt）
- JWT Token 认证，有效期 7 天
- HTTPS 全站加密
- SQL 注入和 XSS 防护
- 文件上传类型和大小限制
- 接口防刷限流

### 3.4 可用性
- 系统可用性 99.9%
- 数据每日备份
- 支持灰度发布

## 4. 技术架构

### 4.1 前端技术栈
- **框架**：Vue 3 + Composition API
- **构建工具**：Vite
- **UI 组件库**：Element Plus / Ant Design Vue
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **HTTP 客户端**：Axios
- **图表**：ECharts
- **样式**：Tailwind CSS + SCSS

### 4.2 后端技术栈
- **语言**：Python 3.10+
- **框架**：FastAPI
- **数据库**：PostgreSQL（主库）+ Redis（缓存/队列）
- **ORM**：SQLAlchemy 2.0
- **迁移**：Alembic
- **任务队列**：Celery + Redis
- **文件存储**：MinIO / 阿里云 OSS
- **文档**：OpenAPI (Swagger)

### 4.3 部署架构
- **容器化**：Docker + Docker Compose
- **反向代理**：Nginx
- **CI/CD**：GitHub Actions
- **监控**：Prometheus + Grafana


## 9. UI设计规范

### 9.1 整体风格
- **设计语言**：企业级专业风格，克制、简洁
- **圆角规范**：统一 4px 小圆角（按钮、输入框、卡片）
- **配色方案**：遵循 Element UI 标准色系
  - 主色：#409eff（蓝色）
  - 成功：#67c23a
  - 警告：#e6a23c
  - 危险：#f56c6c
  - 文字：#303133（主文字）、#606266（次级文字）、#909399（辅助文字）
  - 边框：#dcdfe6、#e4e7ed
  - 背景：#f5f7fa

### 9.2 顶部导航栏
- **高度**：60px
- **布局**：Logo（左）| 导航（中）| 搜索+用户（右）
- **搜索框**：
  - 默认灰色背景（#f5f7fa），无边框
  - Focus 状态：边框高亮（#409eff）+ 2px 阴影
  - 搜索按钮：纯图标（Search icon），hover 时背景变蓝
  - 宽度：180px
- **用户信息**：
  - 头像：36px 圆形，带 1px 边框
  - 用户名：最多显示 8 个字符，超出截断加省略号
  - 默认空用户名显示"用户"
- **导航链接**：
  - 底部 2px 指示线标识当前页面
  - Hover 变色，Active 状态保持高亮

### 9.3 交互细节
- 所有过渡动画：0.2s ease
- 按钮点击反馈：透明度变化或背景色加深
- 下拉菜单：trigger="click"，避免误触

## 10. 部署规范

### 10.1 部署脚本
项目提供标准化部署脚本：

**start.sh** - 启动服务：
```bash
# 功能：
# 1. 检查 frontend/dist 是否存在
# 2. 如不存在，自动执行 npm run build
# 3. 启动 docker-compose 所有服务
./start.sh
```

**stop.sh** - 停止服务：
```bash
# 功能：
# 1. 执行 docker-compose down
# 2. 自动清理 frontend/dist 目录
# 3. 支持所有 docker-compose down 参数（-v, --rmi 等）
./stop.sh
```

### 10.2 构建流程
1. 从 GitHub 拉取代码（dist 目录不会被提交）
2. 运行 `./start.sh` 自动构建前端并启动
3. 生产环境构建使用 Vite production mode
4. Nginx 配置静态资源缓存策略

### 10.3 已知问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面显示 Vue 源码 | Docker volume 挂载覆盖编译后的 dist | 移除 docker-compose.yml 中的 volume 挂载 |
| 404 循环请求 SVG | 删除默认图片但代码仍有引用 | 恢复 default-avatar.svg 和 default-course-cover.svg，或移除 @error 容错逻辑 |
| 从 GitHub 拉取后 403 | dist 目录不存在 | 使用 start.sh 自动构建 |
